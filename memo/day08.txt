CASE문 :

	CASE WHEN 조건식 THEN '참일 때 값' ELSLE '거짓일때 값' END
	
	데이터의 값을 WHEN절의 조건과 차례대로 비교한 후 일치하는 값을 찾아
	THEN 뒤에 있는 결과값을 돌려줍니다.
	
	예)--------------------------------------------
	SELECT * FROM EMP;

	SELECT ENAME "사원명", SAL "급여",
      	CASE
		WHEN SAL >= 3000 THEN 'HIGH'
		WHEN SAL >= 1000 THEN 'MID'
		ELSE 'LOW'
	END "급여등급"
	FROM EMP;
	---------------------------------------------

	CASE문은 전체적으로 하나의 값으로 평가되어 값이 올 수 있는 모든 곳
	에 사용할 수 있습니다.
	
	따라서 SELECT절에서도 사용할 수 있고, WHERE절의 조건문에서도 사용
	할 수 있습니다.

SUB QUERY : 
	특정 SQL문장 안에 또 다른 SQL문장이 포함되어있는것을 말한다.
	여러번 DB접속에 필요한 상황을 한번으로 줄여서 속도를 증가시킬 수 있다.
	
	서브쿼리를 사용할 수 있는 곳

	1) WHERE, HAVING정
	2) SELECT절, DELETE문의 FROM절
	3) UPDATE문의 SET절
	4) INSERT문의 INTO절

[JOIN] :
	여러테이블에 흩어져 있는 정보중 사용자가 필요한 정보만 가져와서 
	가상의 테이블처럼 만들고 결과를 보여주는것.

	조회하려고 하는 테이블들이 너무 많이 쪼개져있으면 속도가 느리기 때문에
	입력, 수정, 삭제의 성능을 향상시키기 위해서 사용된다.

	- 내부조인(INNER JOIN)
	조건이 일치하는 값이 두 테이블에 모두 존대할 때 조회

	- 전제조건
	PK와 FK로 연결이 되어있어야 한다.
	
	테이블A INNER JOIN 테이블B ON 조건식 
	-- INNER은 생략이 가능

	- 등가조인: ON절에 등호가 있을 때
	- 비등가조인 : ON정레 등호가 없을 때 (크다, 작다)
